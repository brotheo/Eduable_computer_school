Project A ¬∑ Gratitude & Wins Journal (Tyler's Assignment)

For students who attended Week 2 (lists & strings) but missed Week 3 (dictionaries & search). Build a menu-driven journal that records daily gratitude with simple in-memory storage.

Console App Lists & Dictionaries User Input Validation Keyword Search

Learning objectives

  • Store structured data in Python dictionaries and keep them inside a list during runtime.
  • Design a friendly text menu and respond to user choices.
  • Filter entries with simple keyword matching.
  • Practice clean, readable output formatting in the terminal.

Checklist before coding

  1. Start a file named gratitude.py and import datetime so you can validate dates.
  2. Create an empty list called entries to hold each journal log as a dictionary.
  3. Plan a menu loop with options to add, view, search, and exit.
  4. Decide how you will format output so the journaling experience feels encouraging.
  5. Test each menu option as you build it‚Äîstay inside the loop until the user chooses Exit.

Step 2 — Collect a new entry

Wrap the input prompts inside a function so the main menu can call it whenever the user selects option 1. Validate the date and make sure the note is not empty before saving the dictionary.

def add_entry():
    raw_date = input("Date (YYYY-MM-DD): ").strip()
    try:
        datetime.strptime(raw_date, "%Y-%m-%d")
    except ValueError:
        print("‚ùå Please use the format YYYY-MM-DD.\n")
        return

    note = input("What are you thankful for today?: ").strip()
    if not note:
        print("‚ùå Please add a few words.\n")
        return

    entries.append({"date": raw_date, "note": note})
    print("‚úÖ Entry saved!\n")
Apply it: Change the prompts and dictionary keys if you are tracking wins, goals, or class takeaways instead of gratitude.

Step 3 — Show the existing log

Step 4 — Search by keyword

Lowercase both sides of the comparison so the search is case-insensitive. A list comprehension keeps the code short and readable.

def search_entries():
    keyword = input("Keyword to search for: ").strip().lower()
    if not keyword:
        print("‚ùå Try typing a word to search for.\n")
        return

    matches = [entry for entry in entries if keyword in entry["note"].lower()]
    if matches:
        print("\nMatches:")
        for entry in matches:
            print(f"- {entry['date']} ‚Äî {entry['note']}")
        print()
    else:
        print("No matching notes yet.\n")
Apply it: Swap the condition if you want to match on mood, tags, or any other field you store in the dictionary.

Project B ¬∑ Smart Shopping Helper (Jade's Assignment)

For students who attended Week 3 (dictionaries & search) but missed Week 2 (loops & lists). Create a tidy shopping helper that stores items in memory, tracks quantities, and stays responsive to user input.

Console UI Lists of Dictionaries Input Validation Sorting & Filtering

Learning objectives

  • Capture structured item data (name + quantity) without relying on files.
  • Reinforce while loops, branching, and list operations.
  • Provide a helpful user experience with confirmations and error checks.
  • Optionally group or sort items to explore bonus list features.

Checklist before coding

  1. Make a new file named shopping_helper.py and plan what data each item needs (name, quantity, optional tags).
  2. Decide how the menu should behave so the user can add, view, update, remove, and exit smoothly.
  3. Sketch validation rules that stop bad inputs (blank names, negative numbers).
  4. Think about how you want to order the items when displaying them.
  5. Write helper functions for repeated logic (finding items, printing the cart) to keep the main loop tidy.

Implementation walkthrough

Step 1 — Create your data structures

Keep everything in memory with a simple list. A helper function makes it easy to find an item by name.

cart = []  # stores dictionaries like {"name": "milk", "quantity": 2}


def find_item(name):
    for item in cart:
        if item["name"] == name:
            return item
    return None
Apply it: Add extra keys (like "category") if your project needs more details‚Äîjust keep using dictionaries.

Step 2 — Add new items with validation

Collect the name and quantity, clean the input, and merge duplicates by reusing find_item.

def add_item():
    name = input("Item name: ").strip().lower()
    if not name:
        print("‚ùå Please enter an item name.\n")
        return

    quantity_raw = input("Quantity: ").strip()
    if not quantity_raw.isdigit() or int(quantity_raw) <= 0:
        print("‚ùå Quantity should be a positive number.\n")
        return

    quantity = int(quantity_raw)
    existing = find_item(name)
    if existing:
        existing["quantity"] += quantity
        print(f"üîÑ Updated {name} to {existing['quantity']}.")
    else:
        cart.append({"name": name, "quantity": quantity})
        print("‚úÖ Added to cart!\n")
Apply it: Replace the prompts if you are planning supplies for class or equipment for sports‚Äîthe validation pattern stays the same.

Step 3 — Display the cart neatly

Sort the items alphabetically so the output is predictable. Track total units to give the user a useful summary.

def view_cart():
    if not cart:
        print("(Your cart is empty.)\n")
        return

    print("\n=== Shopping Cart ===")
    total_units = 0
    for index, item in enumerate(sorted(cart, key=lambda x: x["name"]), start=1):
        print(f"{index}. {item['name'].title()} ‚Äî {item['quantity']} unit(s)")
        total_units += item["quantity"]
    print(f"Total units: {total_units}\n")
Apply it: Switch the ordering or add color coding in the text if you want to highlight priority items.

Step 4 — Update or remove items safely

Re-use the sorted view so the numbering matches what the user just saw. Handle all the ways someone might type invalid input.

def update_item():
    if not cart:
        print("Nothing to update yet!\n")
        return

    view_cart()
    choice = input("Number to update: ").strip()
    if not choice.isdigit():
        print("‚ùå Enter a valid number.\n")
        return

    index = int(choice) - 1
    sorted_cart = sorted(cart, key=lambda x: x["name"])
    if 0 <= index < len(sorted_cart):
        new_qty = input("New quantity: ").strip()
        if new_qty.isdigit() and int(new_qty) > 0:
            sorted_cart[index]["quantity"] = int(new_qty)
            print("üîÑ Quantity updated!\n")
        else:
            print("‚ùå Quantity must be a positive number.\n")
    else:
        print("‚ùå That number is out of range.\n")


def remove_item():
    if not cart:
        print("Nothing to remove yet!\n")
        return

    view_cart()
    choice = input("Number to remove: ").strip()
    if not choice.isdigit():
        print("‚ùå Enter a valid number.\n")
        return

    index = int(choice) - 1
    sorted_cart = sorted(cart, key=lambda x: x["name"])
    if 0 <= index < len(sorted_cart):
        removed = sorted_cart.pop(index)
        cart.clear()
        cart.extend(sorted_cart)
        print(f"üóëÔ∏è Removed {removed['name'].title()} from your cart.\n")
    else:
        print("‚ùå That number is out of range.\n")
Apply it: If your project tracks budgets or deadlines, adjust the confirmation message to reinforce the behavior you want.

Step 5 — Control everything with the main loop

The loop keeps the program running until the user exits. The summary on exit reinforces what they achieved.

def main():
    while True:
        print("\nMenu: (1) Add (2) View (3) Update (4) Remove (5) Exit")
        choice = input("Choose: ").strip()
        if choice == "1":
            add_item()
        elif choice == "2":
            view_cart()
        elif choice == "3":
            update_item()
        elif choice == "4":
            remove_item()
        elif choice == "5":
            total_units = sum(item["quantity"] for item in cart)
            print(f"You planned {len(cart)} item(s) totaling {total_units} unit(s). Happy shopping! üõí")
            break
        else:
            print("Pick 1, 2, 3, 4, or 5.")


if __name__ == "__main__":
    main()
Apply it: Rename menu options if you are scheduling tasks or packing for a trip‚Äîthe branching logic can stay identical.
